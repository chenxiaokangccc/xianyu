# 闲鱼爆款监控部署说明

这个网站不能只部署静态页面，因为真实采集需要 `server.py` 后端接口。推荐部署成 Python Web Service。

## 最简单方案：Render

1. 把整个文件夹上传到 GitHub 仓库。
2. 打开 Render，创建 Web Service。
3. 选择这个 GitHub 仓库。
4. 设置：
   - Runtime: Python
   - Build Command: 留空
   - Start Command: `python3 server.py`
5. 部署完成后，打开 Render 给你的 `onrender.com` 地址。

## Railway 也可以

1. 把整个文件夹上传到 GitHub 仓库。
2. 在 Railway 里创建项目并连接 GitHub 仓库。
3. Start Command 使用：`python3 server.py`
4. 生成公开域名后，用这个域名访问。

## 重要说明

- 不设置密码时，谁拿到网址谁都能打开。
- Cookie 仍然在浏览器本地保存，不会写进代码仓库。
- 商品列表和历史记录现在保存在服务器的 `data/shared-state.json`，两个人打开同一个网址会看到同一份数据。
- 页面会每 4 秒自动同步一次。你添加、删除、采集更新后，对方几秒内会看到变化。
- 如果部署平台重启会清空临时文件，需要开启持久化磁盘，或后续升级成数据库版本。Render 上建议给服务挂载 Persistent Disk，并把 `data` 目录放在持久化磁盘里。
